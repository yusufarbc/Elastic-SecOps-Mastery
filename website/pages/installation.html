Prerequisites

  • Operating System

    Ubuntu 22.04 LTS (Jammy Jellyfish)

  • System Resources

    Minimum: 4 vCPU, 16 GB RAM
    Recommended: 8 vCPU, 32 GB RAM, 1 TB SSD

  • Privileges

    Root or sudo access required for installation.

Quick Start Installation

Run the following commands to clone the repository and start the automated installation script:

git clone https://github.com/yusufarbc/ELK-Ubuntu-Jammy-Build.git
cd ELK-Ubuntu-Jammy-Build
chmod +x scripts/install/elk_setup_ubuntu_jammy.sh
sudo ./scripts/install/elk_setup_ubuntu_jammy.sh

Note: The script will output critical credentials at the end, including the elastic user password and Kibana enrollment token. Save these immediately!

Verification

Validate that all services are running correctly:

1. Check Service Status

systemctl status elasticsearch kibana logstash --no-pager

2. Check Elasticsearch Health

curl -s --cacert /etc/elasticsearch/certs/ca.crt https://localhost:9200 | jq .

3. Verify Logstash Pipelines

sudo /usr/share/logstash/bin/logstash --path.settings /etc/logstash -t

Connecting Log Sources

Windows (WEF)

Winlogbeat on WEC Server

Port: 5045/tcp

Syslog (RFC3164)

Network devices, Linux

Port: 5514/udp+tcp

Syslog (RFC5424)

Modern syslog sources

Port: 5515/tcp

Beats

FortiGate, Filebeat agents

Port: 5044/tcp

Cleanup / Reinstallation

To completely remove all components and data:

sudo systemctl stop logstash kibana elasticsearch || true
sudo rm -rf /etc/elasticsearch /etc/kibana /etc/logstash
sudo rm -rf /var/log/elasticsearch /var/log/logstash
sudo rm -rf /var/lib/elasticsearch /var/lib/logstash
sudo systemctl daemon-reload
