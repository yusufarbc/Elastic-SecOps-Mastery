Elasticsearch Internals

The Inverted Index

At the core is Lucene. Documents are immutable segments. This allows for lock-free reads and efficient kernel page caching.

  • Frame of Reference (FOR): Delta-encoding for compression.
  • PForDelta: Handling outliers in posting lists.
  • Roaring Bitmaps: Efficient filter caching (Sparse vs Dense).

Data Durability

Real-time search without sacrificing durability is achieved via the Translog.

  1. Write to In-Memory Buffer
  2. Append to Translog file
  3. Refresh (1s) -> New Segment (Searching)
  4. Fsync Translog -> Durable on disk

Ingestion Architecture

Logstash Persistent Queues

Protects against data loss during crashes. Uses a sequence of fixed-size page files (Head/Tail pages) and strict checkpointing (ACKs).

Beats & Lumberjack Protocol

Beats use a binary protocol (Lumberjack V2) to communicate with Logstash.

Component Mechanism Benefit
Beats Queue Ring Buffer (Mutex-Guarded) Low GC overhead vs Channels
Backpressure Window Size & ACKs Prevents memory overflow

Security Transformation

XDR Platform

Evolution from passive SIEM to active XDR via the Endgame acquisition.

  • Reflex Engine: Internal decision engine for autonomous response.
  • ECS: Uniform data schema for all sources.
  • EQL: Event Query Language for sequence correlation.
